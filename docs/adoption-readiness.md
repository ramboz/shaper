## A quick readiness check

jig works best when these hold. Treat it as a post-scaffold sanity check:

- [ ] **Git repo health** — initialized, not mid-rebase, you can commit.
- [ ] **Python 3 available** — `python3 --version` works (every jig helper
      and hook is Python 3).
- [ ] **Test command** — you know your stack's test command (pytest /
      vitest / jest are auto-detected; override via a `.jig/test-command`
      file), or you accept there isn't one yet.
- [ ] **Specs/ADRs in-repo** — you'll keep `docs/specs/` and
      `docs/decisions/` under version control as the source of truth for
      "done."
- [ ] **Human review expectations** — the reviewer subagent is an
      *independent compliance check* (a fresh prompt + read-only tools),
      not a replacement for human review, and changes to
      `docs/conventions.md` still want human sign-off.

## Your first 30 minutes

1. **Read the worked example** — your scaffold seeded a `DONE` reference
   spec, [001-adopt-jig](specs/001-adopt-jig/spec.md). Skim it to see the
   shape of a finished slice, then check the
   [status board](specs/README.md).
2. **Set your vision** — run `/jig:vision-elicitation` to fill in
   [product-vision.md](product-vision.md) and
   [architecture.md](architecture.md) (or fill them by hand). This is what
   every later slice is judged against.
3. **Cut your first spec** — `/jig:spec-workflow new <slug>`, then
   SPIDR-split it into thin vertical slices.
4. **Run one slice end to end** — implement → independent review →
   reconcile → DONE. The full lifecycle lives in [workflow.md](workflow.md).

## Is jig paying off?

After two or three slices:

- **Slices are vertical** — each delivered end-to-end value, not a
  half-built layer (the anti-horizontal-phasing guardrail).
- **The reviewer caught something real** — if the compliance pass never
  pushes back, your prompts may be too loose.
- **Memory survives sessions** — a new session picks up from the hot cache
  + `docs/memory/` without a re-briefing.
- **The board reflects reality** — the [status board](specs/README.md)
  shows accurate per-slice state after `workflow.py status-board`.

If those hold, jig is doing its job.
